# 雀魂青云之志自动刷分脚本

1920×1080 分辨率下自动识别牌面、匹配目标牌、点击自摸/跳过的 Python 脚本。

## 原理

1. **截图** — 每轮循环截取牌面区域（90×153）和分数区域（110×30）
2. **ORB 特征匹配** — 对截图提取 ORB 特征点，与预设的目标/干扰模板进行 BFMatcher 交叉匹配，取好匹配数最多的模板
3. **决策** — 最佳模板的匹配数 ≥ 阈值 且 是目标牌 → 点自摸；否则点跳过
4. **分数检测** — Tesseract OCR 读取分数，当前值低于最大值 110/184（≈59.8%）时报警暂停
5. **连点** — 自摸后连点屏幕中心 72 次（0.1s 间隔，共 7.2s）

## 环境要求

- Python ≥3.14
- Windows（使用了 `winsound`、`ctypes`、`keyboard` 等 Windows 特有库）
- Tesseract-OCR 5.4.0+
- 屏幕分辨率 **1920×1080**，游戏窗口全屏

### 安装依赖

安装 Tesseract-OCR：
https://github.com/UB-Mannheim/tesseract/releases

## 使用

```bash
uv run mahjong_auto.py
```

启动后有 10 秒倒计时，切换到游戏窗口即可。**鼠标移到屏幕四角可紧急停止**（pyautogui fail-safe）。

### 控制

| 按键 | 功能 |
|------|------|
| `Ctrl+.` | 切换 暂停 / 恢复 |
| 鼠标移到四角 | 紧急停止（pyautogui.FAILSAFE） |
| 关闭控制台窗口 | 停止程序 |

## 模板配置

### 目录结构

```
pics/
├── targets/       # 目标牌模板（触发自摸）
│   ├── 1m.JPG
│   ├── 1p.JPG
│   ├── 9m.JPG
│   ├── 9p.JPG
│   ├── 9s.JPG
│   └── dong.JPG
└── distractors/   # 干扰牌模板（触发跳过）
    ├── 7p.JPG
    ├── 7s.JPG
    └── 8p.JPG
```

模板为游戏内手动截图（原始像素尺寸，无缩放），ORB 在加载时自动提取特征点。

### 目标牌

`1m`（一萬）、`9m`（九萬）、`9s`（九條）、`1p`（一筒）、`9p`（九筒）、`dong`（東）

## 坐标配置

所有坐标基于 **1920×1080** 全屏：

| 常量 | 坐标 | 说明 |
|------|------|------|
| `TILE_REGION` | (435, 875, 90, 153) | 牌面截图区域 (left, top, width, height) |
| `NUM_REGION` | (365, 805, 110, 30) | 分数数字区域 |
| `CENTER` | (960, 540) | 屏幕中心（连点/鼠标复位） |
| `SELF_BTN` | (1200, 820) | 自摸按钮 |
| `SKIP_BTN` | (500, 950) | 跳过按钮 |

## 阈值说明

| 常量 | 默认值 | 说明 |
|------|--------|------|
| `CONF_TARGET` | 8 | 全局匹配数阈值 |
| `CONF_STRICT` | `{'9s.JPG':10, '1p.JPG':9, '9p.JPG':9, '9m.JPG':5}` | 各个模板的独立阈值 |
| `NUM_ALARM_RATIO` | 110/184 | 分数低于最大值 × 110/184（≈59.8%）时报警 |

阈值是 **好匹配数**（BFMatcher 交叉匹配中 Hamming 距离 < 50 的匹配对数）。模板特征总数越多，理论上可达的匹配数越高，因此特征点偏少的模板（如 9m）需要调低阈值。

## 详细流程

```
┌─────────────┐
│ 加载模板     │  读取 targets/ 和 distractors/ 下的图片，
│ 提取特征点   │  用 ORB 提取特征描述子
└──────┬──────┘
       ↓
┌─────────────┐
│ 10秒倒计时   │  提示切换到游戏窗口
└──────┬──────┘
       ↓
┌─────────────┐  ─── 循环开始 ───
│ 鼠标移至中心 │  (960,540)
├─────────────┤
│ 截取牌面区域 │  90×153
├─────────────┤
│ ORB 特征匹配 │  对所有模板计算好匹配数，
│             │  取最高者
├─────────────┤
│ OCR 读取分数 │  当前值/最大值
│             │  < 110/184 → 报警暂停
├─────────────┤
│ 决策         │  最佳模板是目标 且 匹配数≥阈值
│             │  → 自摸        → 跳过
└──────┬──────┘  └──────┬──────┘
       │                │
       │ 1. 移到自摸按钮  │ 1. 移到跳过按钮
       │ 2. 点击         │ 2. 点击
       │ 3. 连点中心72次  │ 3. 鼠标复位中心
       └────────────────┘
       ↓
    休眠 1.5 秒 → 循环继续
```

## 分数报警

当 OCR 读取到 `cur/max` 格式且 `cur < max × 110/184`（≈59.8%）时：

1. 响三声警报（880Hz）
2. 弹出 Windows 消息框提示
3. 进入暂停状态（`Ctrl+.` 恢复）

OCR 预处理：截图放大 3 倍 + Otsu 二值化。

## 异常处理

主循环包在 `try/except BaseException` 中，任何异常会打印堆栈并 5 秒后自动继续，不会意外退出。`Ctrl+C` 和 `SystemExit` 不受影响。

## 开发备注

- 不要缩放模板图像，保持原始像素尺寸（手动截图时的尺寸）
- ORB nfeatures=500，好匹配距离上限 50
- 使用 `cv2.NORM_HAMMING` + `crossCheck=True` 的 BFMatcher
- 干扰模板（distractors/）中任意图片被判定为最佳时会强制跳过
